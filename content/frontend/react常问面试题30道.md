# React基础概念考点
## 1. 高阶组件（HOC）
- 核心作用：包裹组件增强能力，复用状态或组件功能。
- 业务场景：解决授权、数据管理、错误处理问题。

## 2. 组件选型（类组件vs函数组件）
- 类组件适用：容错、错误补货场景，老项目维护。
- 函数组件优势：React 16+合并部分生命周期，更简洁灵活。

## 3. Props验证与Ref创建
- Props验证：PropTypes或TypeScript（推荐，可解构类型）。
- Ref创建：类组件用createRef（全局唯一），函数组件用useRef（简洁，生命周期内值不变）。

## 4. 组件通信与Context
- 父传子：Props传递数据/函数。
- 子改父：通过父组件传递的函数或全局状态，不可直接修改父组件Props。
- createContext：解决Props穿透问题，通过Provider向下传递数据。

## 5. 缓存相关API（Memo/useMemo/useCallback）
- Memo：缓存组件，避免不必要重渲染（解决对象比较不等问题）。
- useMemo：缓存变量/数据。
- useCallback：缓存函数，配合Memo使用提升性能。


# React生命周期与状态管理
## 1. 生命周期核心考点
- 版本重点：15极少被问，聚焦16-18版本变化。
- 关键要求：牢记生命周期流程图，是面试高频题。

## 2. 状态管理库选型
- 主流库：Redux、MobX、Context、useReducer、Jotai（囧态）。
- 核心问题：状态撕裂（并发模式下共享状态不一致）。
- 解决方案：useTransition、useDeferredValue。

## 3. 状态管理避坑
- 面试禁忌：回答“别人用我也用”“老大定的”，需理解选型逻辑。


# React源码与渲染流程
## 1. 核心渲染流程
- 关键阶段：初始化 → Scheduler → Reconciler → 渲染（Render/Commit）。
- 核心特性：并发渲染模式（Concurrent Model，React 18核心）。

## 2. Fiber架构原理
- 数据结构：Fiber Node（链表结构）。
- 核心机制：双缓存、工作循环（遍历Fiber树）。
- 版本变化：16用时间碎片（Message Channel），现改为微任务/宏任务。

## 3. Scheduler核心要点
- 版本变化：16（实验性，事件分片）→ 17（优化事件系统）→ 18（完整支持并发模式）。
- 批处理：React 18支持任意场景连续更新批处理。

## 4. DOM Diff逻辑
- 区别Vue：无需双指针（Fiber树已排好结构）。
- 比较规则：同级对比，key相同+标签相同则复用，否则不复用；未遍历到的老节点删除。

# React实战与部署考点
## 1. 高频手写题
- 必练：useState、useCallback、useMemo、useLayoutEffect实现。
- 要求：5-8分钟内完成，需熟练掌握底层逻辑。

## 2. 重构与部署
- 开发框架：Nest.js。
- 部署平台：AWS、Cloudflare、阿里云。
- 部署依赖：Cora需布K8S/Docker集群，可搭配Service Worker。

## 3. 服务端渲染（SSR）
- Hybrid适配：老版本需手动调和，React 18自动调和。
- 性能指标：需掌握LCP、FID、CLS，解决首屏问题。
- 核心流程：服务器端渲染组件→脱水数据→客户端注水恢复→React接管节点。

## 4. Serverless渲染
- 关键改变：需适配平台特性（如Cloudflare的轻量级实例环境）。
- 冷启动解决方案：利用V8轻量级隔离环境（如Cloudflare的Isolate），避免Docker启动开销。


# 面试避坑与进阶技巧
## 1. TS类型处理
- 禁忌：避免滥用`any`（会大幅降低面试评分）。
- 解决方案：有经验可猜测类型；查React.d.ts、官方手册；咨询AI或Google。

## 2. 框架对比答题思路
- 核心逻辑：从团队技术一致性、团队技能经验、项目需求灵活性三个维度阐述选型理由。

## 3. Hooks使用禁忌
- 规则：不可写在条件判断/函数体中（破坏链表结构）。
- 特殊场景：需在条件中使用时，需通过组件外状态处理。

## 4. 变态题应对
- 特点：如“Scheduler某行函数的参数/返回值”，考察源码细节。
- 策略：无需强行准备，若目标公司常考此类题，可针对性研读源码关键部分；否则聚焦核心考点。
